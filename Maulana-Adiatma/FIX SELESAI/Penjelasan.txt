Disini aku membuat sistem Dashboard berikut akan ku jelaskan cara kerjanya:

1. User nantinya akan di suguhkan dengan laman login dimana disini aku udh kasih tau apa nama, email dan password mereka secara berurutan (mengantisipasi kalau orng nya lupa sama ketiga faktor itu dan juga mengantisipasi akses ditolak)
2. setelah login user nanti bakal disuguhkan dengan tampilan utama dari dashboard tersebut yang berisi 6 pilihan
    a. jika user memilih kendaraan maka nantinya bakal masuk ke display dari kendaraan itu sendiri berisi (pilihan kendaraan) dan itu bisa diperbarui jika kita sudah mengisi kendaraan tersebut dengan mengikuti arahan dari programnya, jika belum mengisi nantinya bakal keluar pemberitahuan kita belum memilih kendaraan
    b. kalau barang nanti juga diarahkan ke display barang mau tambah atau hapus barang, 
        - jika nambah barang, nanti user bakal disuruh input terus sampe user itu ketik selesai/SELESAI
        - jika hapus nanti tu user bakal ngeliat semua barang yang sudah diinput nah user tinggal menyesuaikan aja maana yang mau dihapus dari barang yang sudah ia input itu
        - barang gk bakal di identifikasi kalau ada salah pengetikan di hapus barang itu
    c. lanjut ke rekomendasi, dimana user nanti akan dibawa ke display dari rekomendasi itu, nambah, hapus dan perbarui rekomendasi
        - jika nambah rekomendasi, nanti user bakal disuruh input kategori rekomen nya tu apaan (film, game, makanan, atau apa) tapi satu kata begitu juga untuk isi (isi dari kategori tadi apaan) satu kata juga.
        - jika hapus rekomendasi, sama kayak barang user bakal ngeliat semua rekomendasi yang sudah di input jadi tinggal sesuaikan aja mana yang pengen dihapus asal sesuaikan huruf besar dan kecilnya
        - jika perbarui rekomendasi, persis di hapus rekomendasi bakal ngeliatin semua rekomendasi namun bedanya jika user memasuki kategori dan isinya nanti bakal disuruh untuk update apa yang ingin diubah dari rekomendasi itu
    d. bagian teman konsep nya sama kayak rekomendasi hanya beda di penamaan nya saja (kalau rekomendasi itu kategori dan isi untuk model nya, kalau teman itu nama dan divisi untuk modelnya)
    e. fitur terakhir itu melihat semua data dari hasil eksekusi yang kita lakukan dari keempat fitur sebelumnya

kekurangan:
1. mudah diakses karna nama, email dan password diberitahukan di laman login :D
2. ketergantungan pada huruf besar dan kecil (soalnya semua fitur aku disini harus sesuai huruf besar dan huruf kecilnya)
3. di fitur rekomendasi dan teman hanya bisa input dengan satu kata
4. belum menerapkan Error handling dan validasi input karena masih belum tahu :D
